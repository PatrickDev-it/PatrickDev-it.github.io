Flagship · Full-stack product engineering

Architecture modernization

AutoBlog CMS

An AI-assisted editorial CMS being rebuilt as one durable product core with explicit identity, authorization, revision history, and failure-safe content operations.

Inspect the repository

Engineering thesis

A credible CMS is a workflow system.

The flagship journey spans authenticated access, workspace-derived policy, drafting, immutable revisions, optimistic concurrency, review, publication, and recovery. AI assistance and media processing are protected external failure domains rather than privileged shortcuts.

The same application logic must serve local review, automated tests, and configured deployment environments.

Target evidence

What the modernization must prove

Product

Deterministic recruiter journey

Seeded content and a short end-to-end workflow without fabricated product behavior.

Security

Session-derived authorization

Explicit RBAC, workspace isolation, negative tests, protected mutations, and constrained uploads.

Data integrity

Revision and concurrency semantics

Immutable revisions, state transitions, conflict detection, migrations, and recovery evidence.

Quality

Accessible, observable operations

Keyboard/focus review, E2E tests, structured failures, health signals, release and rollback paths.

Current boundary

Modernization is visible, not hidden.

  • The public repository represents work in progress; release-grade claims remain gated by executable evidence.
  • Demo shortcuts must not bypass the same domain, persistence, and authorization paths used by configured environments.
  • AI-generated content remains attributable, reviewable, quota-bound, and recoverable.
