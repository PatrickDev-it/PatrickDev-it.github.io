Product · Full stack · Flagship 02

AutoBlog CMS

An AI-assisted editorial CMS being rebuilt around one coherent application core and a verifiable draft-to-publication workflow.

Architecture hardeningPublic repository ↗

Portfolio responsibility

This project demonstrates product judgment and vertical full-stack ownership: polished interaction design supported by real persistence, server-enforced policy, data consistency, failure-safe provider integration, and browser-level evidence.

Flagship workflow

  1. An author creates and autosaves a durable draft.
  2. Meaningful edits create immutable revisions.
  3. A reviewer approves or requests changes through legal state transitions.
  4. An editor schedules or publishes an immutable revision.
  5. Stale concurrent edits surface a conflict instead of overwriting newer work.
  6. AI output remains a suggestion until explicitly applied.

Principal invariant

Visual affordances never substitute for authorization, durable state, or verifiable workflow behavior.

Release gate

Public flagship status requires clean installation, strict quality gates, protected mutations, durable demo data, revision/concurrency E2E coverage, accessibility evidence, and a guided reviewer experience.
