Backend · Reliability · Project 04

Multi-tenant Loyalty Platform

A loyalty, coupon, customer, and gift-card backend being repositioned around its actual domain and hardened for multi-tenant correctness.

Domain modernizationPublic repository ↗

Portfolio responsibility

This project demonstrates backend ownership under concurrency and failure: tenant isolation, strict contracts, transactional state changes, idempotent commands, outbox processing, durable schedules, auditability, and recovery.

Target evidence

  • Session-derived tenant context enforced at policy, query, and persistence boundaries.
  • Validated command schemas without generic mass assignment.
  • Short-lived authentication with explicit rotation and revocation behavior.
  • Idempotent outbox and worker execution under duplicate delivery.
  • Failure-safe email, PDF, birthday, coupon, and gift-card workflows.
  • Strict TypeScript, integration tests, observability, migration, and rollback evidence.

Principal invariant

No request, job, or generated identifier may read or mutate data outside its authenticated tenant boundary.

Current state

The public repository retains its historical name while the implementation is re-scoped. “Electronic invoice” is not used as a product claim for behavior that currently belongs to loyalty operations.
