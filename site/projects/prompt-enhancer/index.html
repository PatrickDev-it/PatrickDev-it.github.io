Prompt Enhancer

A local-first prompt compiler designed to transform underspecified requests into explicit, testable instructions without coupling the product to one model provider.

Engineering thesis

The hard problem is controlled orchestration.

Prompt quality alone is not a systems claim. This project is being structured around stable provider contracts, bounded resources, typed protocol messages, deterministic failure behavior, and a versioned semantic evaluation corpus.

The reviewer path must work without paid APIs or specialized hardware through mock and local execution modes that exercise the same application boundary as configured providers.

Target evidence

What the release must prove

Correctness

Versioned semantic evaluation

A reproducible corpus, baselines, scoring method, failure slices, and regression thresholds.

Architecture

Provider-independent core

One tested contract across deterministic mock, local inference, and explicitly configured remote providers.

Reliability

Bounded real-time protocol

Versioned WebSocket messages, cancellation, reconnect behavior, resource limits, and supervisor failure tests.

Operations

Accessible reviewer runtime

CPU-first quick start plus separately validated hardware profiles, immutable artifacts, smoke tests, and rollback notes.

Current boundary

Claims remain gated.

  • The implementation is under release hardening and is not presented as production-complete.
  • Provider quality and hardware portability will be stated only where repeatable evidence exists.
  • The public repository link will be added after the reviewer journey and evidence package pass their release gates.
